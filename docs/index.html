CCLM is a command-line tool created by Rudolf Müller in C++ for managing C/C++ libraries.

Libraries can be installed from Git repositories or as part of kits (library sets). Prototype written in Python.

POSIX only

Installation

  • Clone the repository: 
    git clone "https://github.com/rudolfmuller/cclm-package-manager.git"
  • Navigate to the installer: 
    cd cclm-package-manager/compiled/dist/linux/
  • Run the installer (may need permission): 
    chmod +x install.sh && ./install.sh
  • All-in-one line: 
    git clone "https://github.com/rudolfmuller/cclm-package-manager.git" && ./cclm-package-manager/dist/linux/install && rm -rf ./cclm-package-manager

Commands

  • purge – Uninstall applications and purge files, Supports -f. 
    cclm purge [-f]
  • template – Creates a project template by name. 
    cclm template <template_name>
  • install – Installs a library. Supports -f. 
    cclm install <user> <repo> [-f]
  • remove – Removes a library. Supports -f. 
    cclm remove <library_name> [-f]
  • connect – Connects CCLM to your project. Supports -a. 
    cclm connect <library_name> [-a]
  • search – Check if repository is available. 
    cclm search <user> <repo>
  • ls – List installed libraries. 
    cclm ls
  • info – Show package information. 
    cclm info <library_name>
  • git – Git command wrapper (for debugging purposes). 
    cclm git <git_args>
  • report – Report a libary. 
    cclm report <library_name>

Structure of library

If you want to create a library, you must add the "info.yaml" file to the first layer of the project, and if it contains dependencies, add the dependencies.txt file with external libraries. If you want a specific format, use the template command. 
CCLM template -h

Installing Kits

Note: Kits are sets of libraries that depend on each other. Names are usually uppercase.

  • Install a kit or library with dependencies:

    CCLM install <library_name> [-f] -d

How it works

CCLM works through a set of commands that manage libraries in a hidden .CCLM folder in your home directory.

  • purge: uninstalls the application and removes all its traces.
  • install: Installs a library.
    • Clones the repository into .CCLM.
    • Checks the repository's first layer for info.yaml and optionally dependencies.txt.
    • Installs dependencies if needed and records metadata (name, version, author, origin, dependencies) in the _sys folder.
  • connect: Links installed libraries to the current project directory.
    • Creates a libs folder.
    • Creates symbolic links for selected packages or all packages.
  • ls: Lists installed libraries using the stored metadata.
  • info: Shows detailed information about a specific package.
  • remove: Removes a library and its record from the .CCLM folder.
  • template: Generates files in the current directory based on the file in .CCLM.
  • search: Checks if a repository exists.
  • git: Serves as a wrapper for git mainly for debugging, arguments passed to CCLM, CCLM passes to git and executes.
  • report: After passing the argument and executing the command, the program combines the arguments into the URL address of the GitHub project CCLM and creates a problem report based on the arguments you entered.
    • Title: The title of the issue.
    • Body: The detailed description of the issue.
    • Assignee: The person responsible for the issue.
    • Labels: Tags to categorize the issue.
